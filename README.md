# Bug Lite - Упрощенный баг-трекер

Проект для курса SecDev (HSE SecDev 2025), представляющий собой API для простого баг-трекера.

## Концепция

Сервис позволяет создавать пользователей и назначать им задачи (issues). Каждая задача имеет заголовок, статус и привязана к пользователю-владельцу.

## Быстрый старт

Для запуска сервиса локально выполните следующие шаги:

```bash
# Создание и активация виртуального окружения
python -m venv .venv
source .venv/bin/activate  # Для Windows: .venv\Scripts\Activate.ps1

# Установка зависимостей
pip install -r requirements.txt -r requirements-dev.txt

# Установка pre-commit хуков
pre-commit install

# Запуск веб-сервера
uvicorn app.main:app --reload
```
Сервер будет доступен по адресу `http://127.0.0.1:8000`.

## Конфигурация и логи

- **Конфигурация**: Основные параметры, такие как хост и порт, могут быть переопределены через переменные окружения (см. `.env.example`).
- **Логи**: Приложение выводит логи в `stdout` в формате JSON. В продуктиве рекомендуется собирать их с помощью специализированных инструментов (Fluentd, Logstash).

## Ритуал перед PR

Перед созданием Pull Request убедитесь, что все проверки проходят:

```bash
# Автоформатирование и линтинг
ruff check --fix .
black .
isort .

# Запуск тестов
pytest -q

# Запуск всех pre-commit хуков
pre-commit run --all-files
```

## Тесты

```bash
# Запуск всех тестов
pytest

# Запуск тестов с отчётом о покрытии
pytest --cov=app
```

## CI/CD

В репозитории настроен CI-конвейер (GitHub Actions), который автоматически запускает проверки при каждом push и pull request в ветку `main`.

## Ключевые эндпойнты

### Аутентификация
- `POST /token`: Получить JWT токен (передать `username` и `password` в form data).

### Пользователи
- `POST /users/`: Создать нового пользователя.
- `GET /users/me`: Получить информацию о текущем пользователе (требует аутентификации).

### Задачи (Issues)
- `POST /issues/`: Создать новую задачу (требует аутентификации).
- `GET /issues/`: Получить список своих задач (требует аутентификации).
- `PATCH /issues/{issue_id}`: Обновить свою задачу (требует аутентификации).
- `DELETE /issues/{issue_id}`: Удалить свою задачу (требует аутентификации).

## Формат ошибок

Все ошибки возвращаются в едином JSON-формате:
```json
{
  "error": {"code": "not_found", "message": "item not found"}
}
```
