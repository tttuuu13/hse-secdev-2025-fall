# Bug Lite - Упрощенный баг-трекер

Проект для курса SecDev (HSE SecDev 2025), представляющий собой API для простого баг-трекера.

## Концепция

Сервис позволяет создавать пользователей и назначать им задачи (issues). Каждая задача имеет заголовок, статус и привязана к пользователю-владельцу. Реализована система ролей: `user` и `admin`. Администраторы имеют доступ ко всем задачам.

## Архитектура

Проект построен на основе слоеной архитектуры:
- **app**: Слой API (FastAPI), отвечает за прием запросов и возврат ответов.
- **services**: Слой бизнес-логики.
- **adapters**: Слой доступа к данным (репозиторий для SQLAlchemy).
- **domain**: Слой доменных моделей (SQLAlchemy).
- **shared**: Общие компоненты, например, кастомные ошибки.

## Быстрый старт

```bash
# Создание и активация виртуального окружения
python -m venv .venv
source .venv/bin/activate  # Для Windows: .venv\Scripts\Activate.ps1

# Установка зависимостей
pip install -r requirements.txt -r requirements-dev.txt

# Установка pre-commit хуков
pre-commit install

# Запуск веб-сервера
uvicorn app.main:app --reload
```
Сервер будет доступен по адресу `http://127.0.0.1:8000`. Документация API (Swagger) - `http://127.0.0.1:8000/docs`.

## Ключевые эндпоинты

Все эндпоинты доступны по префиксу `/api/v1`.

### Аутентификация (упрощенная)
Взаимодействие с API происходит от имени пользователя, чей ID передается в заголовке `X-User-Id`.

Пример запроса:
`curl -X GET "http://127.0.0.1:8000/api/v1/issues/" -H "X-User-Id: 1"`

### Пользователи
- `POST /api/v1/users/`: Создать нового пользователя.

### Задачи (Issues)
- `POST /api/v1/issues/`: Создать новую задачу.
- `GET /api/v1/issues/`: Получить список своих задач (поддерживает пагинацию `?skip=...&limit=...`).
- `GET /api/v1/issues/{issue_id}`: Получить задачу по ID.
- `PATCH /api/v1/issues/{issue_id}`: Обновить свою задачу.
- `DELETE /api/v1/issues/{issue_id}`: Удалить свою задачу.

*Примечание: Пользователи с ролью `admin` имеют доступ ко всем задачам.*

## Формат ошибок

Все ошибки возвращаются в едином JSON-формате:
```json
{
  "code": "KOD_OSHIBKI",
  "message": "Tekst oshibki"
}
```
Примеры кодов: `NOT_FOUND`, `FORBIDDEN`, `CONFLICT`.
