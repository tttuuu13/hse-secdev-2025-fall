# Отчет по харднингу (P12)

## 1. Безопасность контейнера (Dockerfile)
| Что сделано | Результат | Пояснение |
| :--- | :--- | :--- |
| **Уязвимости** | 0 Critical/High | Критических уязвимостей нет благодаря команде `apk upgrade` при сборке. |
| **Линтеры** | Пройдено | Hadolint не выдает ошибок. |
| **Базовый образ** | `python:alpine` | Минимальный размер образа = меньше векторов атаки. |
| **Пользователь** | `appuser` (UID 10001) | Приложение запускается не от root. |

## 2. Безопасность инфраструктуры (Kubernetes IaC)
Файлы Deployment настроены по стандартам безопасности (Checkov):

*   **Минимум привилегий:**
    *   `runAsNonRoot: true` — запрет запуска от root.
    *   `readOnlyRootFilesystem: true` — файловая система закрыта на запись (защита от взлома).
    *   `capabilities.drop: ["ALL"]` — у процесса отобраны все лишние права ядра Linux.
*   **Сетевая защита:** Создана `NetworkPolicy` для ограничения входящего трафика.
*   **Защита токенов:** `automountServiceAccountToken: false` — запрет монтировать токен сервисного аккаунта.
*   **Стабильность:** Добавлены `livenessProbe` и `readinessProbe` для проверки здоровья сервиса.

## 3. Настройка инструментов
*   **Checkov:** Добавлено исключение для правила `CKV_K8S_43`.
*   **Trivy:** Настроен на поиск только критических и высоких уязвимостей.
