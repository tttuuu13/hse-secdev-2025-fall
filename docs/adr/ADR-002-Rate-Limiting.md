# ADR-002-Rate-Limiting: Защита от массового перебора (Rate Limiting)

## Context
Сервис подвержен риску атак массового перебора (brute-force) на эндпоинты аутентификации и регистрации, что может привести к компрометации учетных записей или отказу в обслуживании. Необходимо внедрить механизм ограничения частоты запросов (rate limiting) для защиты этих критически важных эндпоинтов.

## Decision
Принято решение внедрить rate limiting на уровне приложения с использованием библиотеки `fastapi-limiter`. Это позволит гибко настраивать лимиты для различных эндпоинтов, основываясь на IP-адресе клиента. Для эндпоинтов `/api/v1/users` (регистрация) и `/api/v1/auth/login` (вход) будет установлен лимит в 100 запросов в минуту с одного IP-адреса.

## Alternatives
1.  **Rate limiting на уровне прокси/балансировщика**:
    *   **Плюсы**: Снимает нагрузку с приложения, централизованное управление.
    *   **Минусы**: Требует дополнительной инфраструктуры (например, Nginx, Cloudflare), может быть сложнее в настройке для специфических эндпоинтов.
2.  **Ручная реализация в коде**:
    *   **Плюсы**: Полный контроль над логикой.
    *   **Минусы**: Высокая сложность, подверженность ошибкам, необходимость поддержки собственного решения.

## Consequences
*   **Положительные**:
    *   Защита от атак массового перебора.
    *   Повышение стабильности и доступности сервиса.
    *   Снижение нагрузки на сервер при злонамеренных действиях.
*   **Отрицательные**:
    *   Небольшое увеличение сложности кодовой базы.
    *   Необходимость настройки и мониторинга лимитов.

## Security Impact
*   **Защита от Brute-Force**: Предотвращает попытки подбора паролей и учетных записей.
*   **Отказ в обслуживании**: Снижает риск DoS-атак, направленных на перегрузку эндпоинтов аутентификации.

## Rollout Plan
1.  Установить `fastapi-limiter` и настроить его с Redis.
2.  Применить декораторы `@limiter.limit` к соответствующим эндпоинтам в `app.main`.
3.  Добавить интеграционные тесты для проверки работы rate limiting.
4.  Развернуть изменения, мониторя логи на предмет ложных срабатываний.
