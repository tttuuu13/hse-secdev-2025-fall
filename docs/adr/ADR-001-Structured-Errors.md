# ADR-001-Structured-Errors: Единый формат ошибок API (RFC 7807)

## Context
В текущей реализации API ошибки возвращаются в различных форматах, что затрудняет их обработку на стороне клиента и мониторинг. Необходимо стандартизировать формат ответов об ошибках для улучшения взаимодействия с клиентами API и упрощения отладки.

## Decision
Принято решение использовать формат ошибок, соответствующий RFC 7807 (Problem Details for HTTP APIs). Это позволит возвращать стандартизированные JSON-объекты с деталями проблемы, включая `type`, `title`, `status`, `detail` и `instance`. Дополнительно будет добавлен `correlation_id` для трассировки запросов.

## Alternatives
1.  **Простой JSON-формат**: `{"message": "..."}`.
    *   **Плюсы**: Простота реализации.
    *   **Минусы**: Недостаточно информации для клиента, отсутствие стандартизации, затрудняет автоматическую обработку ошибок.
2.  **Использование стандартных HTTP-статусов без тела ответа**:
    *   **Плюсы**: Максимальная простота.
    *   **Минусы**: Отсутствие какой-либо информации об ошибке, невозможность предоставить детали.

## Consequences
*   **Положительные**:
    *   Улучшенная обрабатываемость ошибок на стороне клиента.
    *   Единый подход к логированию и мониторингу ошибок.
    *   Упрощение отладки и поддержки API.
    *   Соответствие стандарту RFC 7807.
*   **Отрицательные**:
    *   Необходимость модификации существующих обработчиков ошибок.
    *   Потенциальное увеличение размера ответов при ошибках.

## Security Impact
*   **Маскирование деталей**: В production-среде детали ошибок будут маскироваться, чтобы предотвратить утечку чувствительной информации.
*   **Correlation ID**: Позволит эффективно отслеживать запросы и связанные с ними ошибки, что важно для расследования инцидентов безопасности.

## Rollout Plan
1.  Разработать централизованный обработчик исключений в `app.main`.
2.  Интегрировать его для всех типов ошибок (4xx, 5xx).
3.  Обновить существующие тесты и добавить новые для проверки формата ошибок.
4.  Развернуть изменения с использованием feature-флага или canary-развертывания.
