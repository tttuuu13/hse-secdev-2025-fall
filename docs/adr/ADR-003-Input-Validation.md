# ADR-003-Input-Validation: Строгая валидация входных данных

## Context
Сервис обрабатывает данные, поступающие от внешних клиентов, что создает риск атак, связанных с некорректными или вредоносными входными данными (например, SQL-инъекции, XSS, переполнение буфера). Отсутствие строгой валидации может привести к уязвимостям и нестабильности приложения.

## Decision
Принято решение использовать Pydantic для строгой валидации всех входных данных на уровне API. Pydantic позволяет определять схемы данных с указанием типов, ограничений и правил валидации, что обеспечивает автоматическую проверку входящих запросов. Это соответствует принципу "никогда не доверяй вводу пользователя".

## Alternatives
1.  **Ручная валидация в контроллерах/сервисах**:
    *   **Плюсы**: Полный контроль над логикой валидации.
    *   **Минусы**: Дублирование кода, высокая вероятность ошибок, сложность поддержки, неконсистентность.
2.  **Валидация на уровне базы данных**:
    *   **Плюсы**: Обеспечивает целостность данных на самом низком уровне.
    *   **Минусы**: Не защищает от атак на уровне приложения до сохранения данных, ошибки проявляются поздно, менее информативны для клиента.

## Consequences
*   **Положительные**:
    *   Значительное снижение риска уязвимостей, связанных с некорректным вводом.
    *   Улучшение качества и надежности данных.
    *   Упрощение разработки за счет автоматической валидации.
    *   Более информативные сообщения об ошибках для клиентов API.
*   **Отрицательные**:
    *   Необходимость определения Pydantic-схем для всех входных данных.
    *   Небольшое увеличение времени обработки запросов из-за валидации.

## Security Impact
*   **Предотвращение инъекций**: Защита от SQL-инъекций, XSS и других атак, использующих вредоносные данные.
*   **Целостность данных**: Гарантия того, что в приложение поступают только корректные данные.
*   **Отказ в обслуживании**: Снижение риска переполнения буфера или других атак, направленных на нестабильность приложения.

## Rollout Plan
1.  Проверить все существующие эндпоинты на предмет использования Pydantic-схем для входных данных.
2.  Добавить или обновить Pydantic-схемы в `app.schemas` для всех критически важных эндпоинтов.
3.  Добавить unit-тесты для Pydantic-схем с использованием валидных и невалидных данных.
4.  Добавить интеграционные тесты для проверки ответов API при невалидных входных данных.
