# NFR.md — Таблица требований безопасности

| ID | Название | Описание | Метрика/Порог | Проверка (чем/где) | Компонент | Приоритет |
|---|---|---|---|---|---|---|
| NFR-01 | Хеширование паролей | Пароли пользователей должны храниться в виде хешей, созданных с использованием современного и стойкого алгоритма. | Алгоритм: bcrypt | Проверка кода + Unit-тест | `app.services.user_service` | High |
| NFR-02 | Авторизация на основе владения | Пользователь должен иметь доступ (чтение, изменение, удаление) только к своим ресурсам (задачам). | 100% эндпоинтов CRUD | Интеграционные тесты (IDOR) | `app.services.issue_service` | High |
| NFR-03 | Права администратора | Пользователь с ролью `admin` имеет полный доступ ко всем ресурсам в обход проверки на владельца. | 100% эндпоинтов CRUD | Интеграционные тесты | `app.services.issue_service` | High |
| NFR-04 | Структурированные ошибки | Все ошибки API (4xx, 5xx) должны возвращаться в едином формате `{"code": "...", "message": "..."}`. | 100% ответов с ошибками | Контрактные тесты | `app.main` (exception handler) | Medium |
| NFR-05 | Защита от массового перебора | Ограничить количество запросов к эндпоинтам создания пользователя и входа в систему. | Не более 100 запросов в минуту с одного IP | Нагрузочный/rate-limit тест | `app.main` (middleware) | Medium |
| NFR-06 | Валидация входных данных | Все данные, поступающие от клиента, должны проходить строгую валидацию на уровне API. | 100% эндпоинтов с телом запроса | Unit-тесты со невалидными данными | `app.schemas` (Pydantic) | High |
| NFR-07 | Отсутствие уязвимостей в зависимостях | В проекте не должно быть зависимостей с известными уязвимостями уровня High или Critical. | 0 High/Critical уязвимостей | Сканирование зависимостей в CI (например, `pip-audit`) | `requirements.txt` | High |
| NFR-08 | Производительность API | 95-й перцентиль времени ответа ключевых эндпоинтов (`GET /issues`) не должен превышать 200 мс. | p95 <= 200ms | Нагрузочные тесты | `app.services.issue_service` | Medium |
