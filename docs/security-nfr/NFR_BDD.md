# NFR_BDD.md — Приёмка в формате Gherkin

### Feature: Авторизация на основе владения ресурсом

**Scenario:** Пользователь не может получить доступ к чужой задаче
  **Given** существуют два пользователя, "UserA" и "UserB"
  **And** "UserA" является владельцем задачи с ID "issue-123"
  **When** "UserB" отправляет GET-запрос на `/api/v1/issues/issue-123`
  **Then** сервис должен вернуть ответ со статусом `403 Forbidden`
  **And** тело ответа должно содержать JSON `{"code": "FORBIDDEN", "message": "Not enough permissions"}`

### Feature: Права администратора

**Scenario:** Администратор может получить доступ к любой задаче
  **Given** существует пользователь "UserA", который владеет задачей с ID "issue-123"
  **And** существует пользователь "Admin" с ролью `admin`
  **When** "Admin" отправляет GET-запрос на `/api/v1/issues/issue-123`
  **Then** сервис должен вернуть ответ со статусом `200 OK`
  **And** тело ответа должно содержать полную информацию о задаче "issue-123"

### Feature: Структурированные ошибки API

**Scenario:** Сервис возвращает ошибку в едином формате при отсутствии ресурса
  **Given** не существует задачи с ID "non-existent-issue"
  **And** существует аутентифицированный пользователь "UserA"
  **When** "UserA" отправляет GET-запрос на `/api/v1/issues/non-existent-issue`
  **Then** сервис должен вернуть ответ со статусом `404 Not Found`
  **And** тело ответа должно содержать JSON `{"code": "NOT_FOUND", "message": "Issue not found"}`
