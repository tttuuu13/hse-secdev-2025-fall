# STRIDE — Угрозы и контроли

| Поток/Элемент | Угроза (STRIDE) | Описание угрозы | Контроль | Ссылка на NFR |
|---|---|---|---|---|
| F1, F2 | **S**poofing (Подмена) | Пользователь подделывает заголовок `X-User-Id`, чтобы выдать себя за другого пользователя. | Внедрение полноценной аутентификации на основе токенов. | NFR-01 |
| F1, F2 | **D**enial of Service (Отказ в обслуживании) | Злоумышленник отправляет огромное количество запросов, перегружая сервис. | Ограничение частоты запросов (Rate Limiting). | NFR-05 |
| F1 | **E**levation of Privilege (Повышение привилегий) | Пользователь при регистрации пытается установить себе роль `admin`, передав ее в теле запроса. | Роль назначается только на сервере и не может быть изменена клиентом. | NFR-03 |
| F2 | **E**levation of Privilege (IDOR) | Пользователь пытается получить/изменить/удалить чужую задачу, подставляя ее ID в URL. | Проверка владения ресурсом на уровне сервиса для каждого запроса. | NFR-02 |
| P1 (API) | **I**nformation Disclosure (Раскрытие информации) | В ответах об ошибках раскрывается внутренняя информация (например, stack trace). | Глобальный обработчик исключений, возвращающий структурированные и общие ошибки. | NFR-04 |
| P1 (API) | **D**enial of Service (Атака на парсер) | Злоумышленник отправляет специально сформированный, очень большой или сложный JSON, чтобы вызвать сбой. | Использование Pydantic с валидацией и ограничениями на длину полей. | NFR-06 |
| F3, F5, F6 | **T**ampering (Фальсификация) | Злоумышленник пытается выполнить SQL-инъекцию через поля ввода. | Использование ORM (SQLAlchemy), которая автоматически параметризует запросы. | NFR-06 |
| DS1 (БД) | **I**nformation Disclosure (Раскрытие информации) | Злоумышленник получает доступ к файлу БД и видит пароли пользователей в открытом виде. | Хеширование паролей перед сохранением в БД. | NFR-01 |
| Проект | **E**levation of Privilege (Уязвимые зависимости) | Злоумышленник эксплуатирует известную уязвимость в одной из библиотек проекта. | Регулярное сканирование зависимостей на уязвимости. | NFR-07 |
| F1, F2 | **T**ampering (Фальсификация данных в пути) | Злоумышленник перехватывает и изменяет данные между клиентом и сервером. | Использование HTTPS для шифрования всего трафика. | - |
| P1 (API) | **R**epudiation (Невозможность отказа) | Пользователь отрицает, что он создал или изменил задачу, так как действия не логируются. | Внедрение аудируемого логирования для всех операций, изменяющих данные. | - |
| DS1 (БД) | **D**enial of Service (Медленные запросы) | Запросы к БД выполняются медленно из-за отсутствия индексов на часто запрашиваемых полях. | Добавление индексов в модели SQLAlchemy для полей `email`, `username`, `title`. | NFR-08 |
