# Check-list Code Review (Bug Lite)

## Общие проверки
- [ ] Объём PR разумный; цель понятна (одна тема на PR).
- [ ] Именование и структура кода понятны и соответствуют домену (например, `issue`, `title`).
- [ ] Документация/комментарии (если есть) релевантны и точны.

## Функциональность
- [ ] Изменения покрыты тестами (позитивные и негативные сценарии).
- [ ] Обработка ошибок и граничных случаев (например, пустой `title`, несуществующий `issue_id`).
- [ ] Эндпоинты соответствуют спецификации (правильные пути, методы, параметры).

## Специфика "Bug Lite"
- [ ] Логика работы с `issue` корректна (создание, получение, листинг).
